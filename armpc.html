Intro

The goal of this project was to gain a strong understanding of ARMv8 assembly language and ARM Architecture. The main challenges for me was understanding
different instructions, their purpose, and how addressing memory worked with certain instructions.

Testbenches

Here I have included the terminal verifying the results for both tests. The testbench here just gives the processor a start address, reset signal, and clock signal. From
there it just reads the current address in the instruction memory. The processor then reads the instructions from the instruction memory and has to correctly perform
the assembly code. If performed correctly, the information read from data memory will be correct. Because both tests passed, we know that the processor functions
as intended.
